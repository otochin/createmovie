# 動画検索機能 - 仕様書

## 1. 概要

### 1.1 目的
左ナビゲーションに「動画検索」を追加し、**ここ最近再生数が伸びているYouTube動画を検索して、サムネイル画像を参考にできるようにする**ことを目的とする。  
台本・画像生成のインスピレーション源として、トレンドや人気動画のサムネイルを確認できるようにする。

### 1.2 位置づけ
- **配置**: 左サイドバーのナビゲーションに「🔍 動画検索」を追加（ホーム・台本生成・音声生成・画像生成・動画編集と同列）。
- **ワークフロー**: 動画制作の前段階の「リサーチ」として利用。台本生成や画像生成の前に、キーワード・時期で絞り込んで人気動画を確認し、サムネイルを参考にできる。

### 1.3 実行環境・技術
- **既存環境**: 本プロジェクトの Streamlit アプリ内の1ページとして実装。
- **外部API**: YouTube Data API v3（検索・動画情報・サムネイルURLの取得）。APIキーは環境変数で管理（`.env`）。

---

## 2. 機能要件

### 2.1 検索機能

#### キーワード検索
- **入力**: ユーザーがキーワード（テキスト）を入力。
- **処理**: YouTube Data API v3 の `search.list` で `q` にキーワードを渡して検索。
- **備考**: 日本語キーワード対応。部分一致・関連動画の検索はAPIの仕様に準拠。

#### 公開時期の絞り込み
- **入力**: 公開された時期で絞り込む条件を指定する。
- **案（いずれかまたは組み合わせで仕様確定）**:
  - **プリセット**: 「直近24時間」「直近7日間」「直近30日間」「直近3ヶ月」などから選択。
  - **日付範囲**: 開始日・終了日をカレンダー（または日付入力）で指定。
- **処理**: `search.list` の `publishedAfter`（および必要に応じて `publishedBefore`）でAPIに渡す。
- **デフォルト**: 未指定の場合は「直近30日間」など、トレンドを見るのに適した初期値を推奨。

#### その他の絞り込み（任意・将来拡張）
- 動画の長さ（ショートのみ / 通常動画のみ）、並び順（関連度 / 再生数 / 公開日）などは仕様確定時に追加可能。

### 2.2 検索結果の表示

#### 一覧表示
- **絞り込み**: 検索結果は**概要（description）が存在する動画のみ**を表示する。`videos.list` で取得した `snippet.description` が空または空白のみの動画は一覧から除外する。結果メッセージに「概要がある動画のみ表示」と明記する。
- **表示項目**（1動画あたり）:
  - **サムネイル画像**: 参考用。APIが返す `thumbnails` の高解像度URL（`high` または `maxres`）を表示。
  - **タイトル**: 動画タイトル。
  - **チャンネル名**: 投稿者チャンネル名。
  - **公開日**: 公開日時（YYYY-MM-DD または 相対表示「○日前」など）。
  - **再生数**: 再生回数（取得可能な場合。`videos.list` で statistics を取得）。
- **レイアウト**: サムネイルを前面にしたカード形式（グリッドまたはリスト）。スクロールで複数件表示。
- **件数**: APIの1リクエストあたりの上限（最大50件）を考慮し、初期は「最大25件」などで制限し、必要に応じてページネーションや「もっと見る」で拡張可能。

#### サムネイルの「参考」としての利用
- **表示**: 検索結果のサムネイルはあくまで「参考表示」とする（クリックで拡大表示は任意）。
- **画像生成との連携（将来拡張）**: 気になったサムネイルを「参考画像」として画像生成ページに渡す機能（URLのコピー、または「参考画像に追加」で `output/` に保存して画像生成で利用）は、Phase 2 以降で検討可能。初版では「検索・一覧表示・サムネイル確認」に留めてもよい。

#### 動画へのリンク
- 各結果からYouTubeの動画ページへ遷移できるリンク（タイトルまたは「YouTubeで見る」ボタン）を設置する。

### 2.3 UI・UX

- **ページ構成**:
  1. 上部: キーワード入力（テキストボックス）、公開時期の絞り込み（セレクトボックスまたは日付範囲）、検索ボタン。
  2. その下: 検索結果エリア（サムネイル＋メタ情報の一覧）。
- **エラーハンドリング**:
  - APIキー未設定時はメッセージを表示し、`.env` の設定を案内。
  - APIエラー（クォータ超過、ネットワークエラー等）はユーザーに分かりやすいメッセージで表示。
- **ローディング**: 検索実行中はスピナーまたはプログレス表示を行う。

### 2.4 APIキー・設定

- **YouTube Data API v3**:
  - 環境変数: `YOUTUBE_API_KEY` または `GOOGLE_API_KEY`（プロジェクトで既存の命名に合わせる）。
  - `.env.example` に追加し、`ENV_SETUP.md` に取得手順を追記する。
- **クォータ**: YouTube Data API は1日あたりのクォータがある。検索は `search.list` で 100 ユニット/回、`videos.list` で 1 ユニット/回程度。大量検索時はクォータ考慮が必要。

## 3. 技術仕様

### 3.1 使用API

| API | 用途 | 主なパラメータ |
|-----|------|----------------|
| YouTube Data API v3 `search.list` | キーワード・時期で動画を検索 | `q`, `publishedAfter`, `publishedBefore`, `type=video`, `part=snippet`, `order`（任意） |
| YouTube Data API v3 `videos.list` | 動画詳細・再生数取得 | `id`, `part=snippet,statistics` |

- **認証**: APIキー認証（OAuth は不要。公開データの読み取りのみ）。
- **レスポンス**: サムネイルURLは `snippet.thumbnails` の `high.url` または `maxres.url` を使用。

### 3.2 公開時期の変換

- 「直近7日間」→ `publishedAfter`: 現在日時 - 7日（ISO 8601形式）。
- 「直近30日間」→ 同様に 30 日前の日時を算出。
- 日付範囲指定の場合は、開始日 00:00:00 を `publishedAfter`、終了日 23:59:59 を `publishedBefore` に渡す。

### 3.3 データ形式・保存

- **初版**: 検索結果はAPIから取得してその場で表示し、永続保存はしない（キャッシュは任意）。
- **将来**: お気に入りサムネイルの保存や、画像生成への「参考画像」引き渡しを行う場合は、`output/` 配下やセッション状態で管理を検討。

## 4. ディレクトリ・ファイル構成の追加案

```
createmovie/
├── main.py                     # サイドバーに「🔍 動画検索」を追加し、video_search_page を呼び出し
├── config/
│   ├── config.py               # （必要に応じて）YouTube APIキー読み込みを追加
│   └── constants.py             # （必要に応じて）検索プリセット期間の定数
├── ui/pages/
│   ├── video_search_page.py    # 動画検索画面（新規）
│   └── ...
├── utils/ または video/ または新規モジュール
│   └── youtube_client.py       # YouTube Data API v3 の検索・動画情報取得（新規）
└── docs/
    ├── SPEC_VIDEO_SEARCH.md    # 本仕様書
    ├── README_PLAN.md          # 2.6 動画検索機能への言及を追加
    └── ENV_SETUP.md            # YouTube API キー設定の説明を追加
```

- **youtube_client.py**: `search_videos(keyword, published_after=None, published_before=None, max_results=25)` のようなインターフェースで、`search.list` と必要に応じて `videos.list` を組み合わせ、サムネイルURL・タイトル・チャンネル名・公開日・再生数を返す。
- **video_search_page.py**: Streamlit の `st.text_input`, `st.selectbox`（時期）, `st.button` で検索フォームを構成し、結果を `st.columns` やカード風レイアウトで表示。

## 5. セキュリティ・運用

- **APIキー**: `.env` に保持し、リポジトリにコミットしない（`.gitignore` 済み）。
- **クォータ**: 開発・検証時は少なめの `max_results` から始め、本番利用時に増やす場合はYouTube Cloud コンソールのクォータを確認する。
- **ログ**: API呼び出しのエラーログは `utils.logger` で記録（APIキーやトークンはログに含めない）。
